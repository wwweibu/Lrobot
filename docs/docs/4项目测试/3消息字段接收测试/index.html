测试结果​

  • 1.1
    • 文字+换行+文字+超级表情+小黄脸表情+emoji表情
    • LR5921 会以文本类型解析 emoji 表情
    • LR5921 表情字段中包含 id,raw,resultId,chainCount
  • 1.2
    • 文字+换行+文字+超级表情+小黄脸表情+emoji表情
    • LR232 会以文本类型解析 emoji 表情
    • LR232 表情字段中包含 id,type,ext
  • 1.3
    • 文字+at+文字
    • at 时会自动生成一个空格
    • LR232 无法解析 at
  • 1.4
    • 超级表情（单独）
    • 超级表情单独发送与一起发送无任何区别
    • LR232 需要 at 无超级表情
  • 2.1
    • 猜拳/骰子（单独）
  • 2.2
    • 文字+猜拳+骰子
    • 猜拳和掷骰子混在其他消息一起发送时，LR5921 可以获取掷骰子的点数，而不能获取猜拳的结果
  • 2.3
    • 猜拳/骰子（单独）
    • 猜拳和掷骰子时，LR232 无法解析结果
    • 可得知骰子表情为 358 号表情，猜拳表情为 359 号表情
  • 3.1
    • 回复（长按消息选择引用）
    • 引用消息时会自动添加 @ 某人，但不是必须，@ 可以删掉
    • 回复时双层回复无法获取
    • LR232 无法解析回复
  • 4.1
    • 合并转发消息
    • 单转发消息无法识别
    • 转发里面套回复同样无法获取（转发前后被回复消息均无法获取，故无法获取是没有问题的）
    • LR232 无法解析转发消息（但 msg_type=3,一般是 0）
  • 5.1
    • 电脑抖动/其他类型
    • 戳戳中，戳一戳=电脑抖动，且其他的都会以“[xxx]请使用最新版本手机QQ查看。”的形式发送；右边口袋的内容无法解析
    • LR232 无法发送抖动消息
  • 6.1
    • 商城表情/收藏里的商城表情
    • 当商城表情变成收藏表情时，LR5921 能识别成商城表情
  • 6.2
    • 商城表情/收藏里的商城表情
    • LR232 可以接收商城表情但无法识别
    • 当商城表情变成收藏表情时，LR232 能识别成商城表情
  • 6.3
    • 收藏里的自定义表情
  • 6.4
    • 收藏里的自定义表情
    • LR232 识别到自定义表情跟图片一致，故不测图片
  • 6.5
    • 图片/拍照
    • LR5921 的拍照与发送图片相同
    • LR232 图片字段包括 file_size,width,height
    • LR5921 图片字段包括 sub_type,file_size
    • 观察可得知，LR5921 和 LR232 的文件名是一样的
  • 6.6
    • 文字+动画表情（代码格式[Hurt]以及从表情列表中选择）
    • WECHAT 无法接收收藏表情及商城表情
    • WECHAT 会以文本类型解析 emoji 表情
  • 6.7
    • 图片
    • 相册中发送/截图
  • 6.8
    • 文字+普通表情+商城表情+emoji 表情+伪造表情
    • BILI 可以伪造表情（见消息字段解析）
    • BILI 的颜文字表情会以文字形式发送
    • BILI 会以文本类型解析 emoji 表情
  • 6.9
    • 图片
    • 只能单发文件
  • 7.1
  • 7.2
  • 7.3
    • 以语音格式长按输入（手机）
    • 分享音乐（qq 音乐）无法接收
  • 8.1
    • LR5921 只有长按拍照才发的是视频，不然是文件
    • LR23 无法长按拍照
  • 8.2

  • 9.1
